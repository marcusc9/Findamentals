Build the model so the decision survives it.

A financial model is not just an Excel file. It is a decision system. The aim is to make the assumptions visible, the calculations consistent, the outputs testable, and the conclusion robust enough to survive a changed input.

Good modelling is structure, judgement, and checks.

Start with architecture before formulas. A usable model separates inputs, calculations, outputs, checks, and scenarios. Inputs are the assumptions a decision-maker can debate. Calculations translate those assumptions into mechanics. Outputs answer the question. Checks tell you whether the workbook can be trusted. Scenarios show how fragile the answer is.

Take a small subscription software business. Customers at the start of the month, new customers, churn, average revenue per account, gross margin, fixed staff costs, marketing spend, and working-capital lag are the drivers. The core equations are plain: revenue = customers x average revenue per account, EBITDA = revenue - variable costs - fixed costs, and closing debt = opening debt + drawdowns - repayments. If the model cannot explain those bridges clearly, clever formulas are just hiding risk.

Practical Excel discipline helps. Use consistent time columns. Keep copy-across formulas short where possible. Use XLOOKUP or INDEX/MATCH to pull assumptions cleanly, SUMIFS for grouped data, EOMONTH for period logic, IF or IFS for flags, and LET when a formula needs named intermediate steps. Scenario switches should change assumptions, not overwrite formulas.

Then build the audit trail. A balance-sheet check should equal zero. Cash flow should tie to the movement in cash. Sign conventions should be consistent. Hardcodes inside calculation blocks should be easy to spot. Formula consistency should be testable across rows. A sensitivity table should make commercial sense, not merely recalculate. The point is not to make the model impressive. It is to make the decision less brittle.
